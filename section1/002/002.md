# 문제

삼각형 판별하기
길이가 서로 다른 A, B, C 세 개의 막대 길이가 주어지면 이 세 막대로 삼각형을 만들 수 있
으면 “YES"를 출력하고, 만들 수 없으면 ”NO"를 출력한다.
▣ 입력설명
첫 번째 줄에 100이하의 서로 다른 A, B, C 막대의 길이가 주어진다.
▣ 출력설명
첫 번째 줄에 “YES", "NO"를 출력한다.
▣ 입력예제 1
6 7 11
▣ 출력예제 1
YES
▣ 입력예제 1
13 33 17
▣ 출력예제 1
NO

## 설계

### 큰 수 찾기

- 제일 큰수를 고르자. -> 이때 O(N)으로 해결해보자.

1. 특정 변수에 특정 target 저장
2. 나머지를 순환하는데 만약 target < currentNum 일경우 -> target을 새로 업데이트 한다.
3. 순환이 종료되면 최종 target이 최대 값이 된다.

### 최댓값을 이용한 삼각형 판별

- 이제 지금 알고 있는 정보는 두개다.

  1. 모든 수의 합
  2. 최댓값

- 알고 있는 정보만 활용해보자.
  - 모든수의 합에서 최댓값을 뺀다. -> 이게 나머지 수들의 합이다.
  - 나머지 수들의 합이 최댓값보다 커야 삼각형 조건이 완성된다.

```javascript
function solution(...nums) {
  let maxNum = nums[0];
  let maxIdx;
  nums.forEach((currentNum, idx) => {
    if (maxNum < currentNum) {
      maxNum = currentNum;
      maxIdx = idx;
    }
  });
  nums.splice(maxIdx, 1);

  const restSum = nums.reduce((acc, num) => {
    return acc + num;
  }, 0);
  if (restSum > maxNum) {
    console.log("YES");
  } else {
    console.log("NO");
  }
}
```

-> O(N)이긴 하지만... 단한번의 순회로 끝낼 수 없을까?
-> splice가 굳이 필요할까?

## 개선 \_ O(n) 3번을 -> O(n) 1번으로 줄이기.

처음에는 splice를 통해 최댓값인 수를 제외하고, 남은 수들을 reduce를 통해 더했다. 하지만 이 과정은 전체 합에서 최댓값을 제거해서 구할 수 있다.
하지만 단 한번의 reduce로 최댓값과 나머지 수들의 합을 구해보자.

```javascript
function solution(...nums) {
  let maxNum = 0;

  const restSum = nums.reduce((acc, currentNum) => {
    if (maxNum < currentNum) {
      acc += maxNum; // 더하지 못한 값 더해주기

      maxNum = currentNum; //최댓값 업데이트

      return acc;
    }

    return acc + currentNum;
  }, 0);

  if (restSum > maxNum) {
    console.log("YES");
  } else {
    console.log("NO");
  }
}

console.log(solution(6, 7, 11));
```

- 순환중에 새로운 큰 값을 만나면 기존에 더하지 못했던 과거의 최댓값을 acc에 더해줌으로써 최댓값을 제외한 나머지 수들의 합을 구했다.
