## 1차 풀이

indexOf를 생각하지 못하고 이중 for문으로 풀이함.
루프를 돌면서 1차 루프의 대상(base)과 2차 루프의 대상(target)이 만나면 flag로 대체하는 방식으로 전개. -> 원본 배열의 길이를 유지하기 위해 flag로 대체함.

그리고 마지막에 flag를 필터링해 결과로 반환.

```javascript
function solution(s) {
  let arr = [...s];
  const flag = 1;
  for (let i = 0; i < arr.length - 1; i++) {
    const base = arr[i];
    if (base === flag) continue;

    for (let n = i + 1; n < arr.length; n++) {
      // 큰실수... n을 루프마다 계속 1로 초기화해서 이중 for문이 정상작동 X
      const target = arr[n];
      if (target === flag) continue;
      if (base === target) arr[n] = flag;
    }
  }

  const final = arr.filter((n) => n !== flag);
  return final.join("");
}
```

## 2차 풀이\_indexOf

중복된 문자열을 제거하는 문제다.
아래의 과정을 문자열 길이만큼 순환한다. {

indexOf의 특징을 이용해서 중복을 제거해보자.(indexOf는 입력한 문자열과 동일한 첫번째 요소를 반환한다.)
입력한 요소의 idx와 indexOf의 반환값이 동일한 경우에만 반환값에 중첩한다.

}

이 문제 풀때 중요한 점은 indexOf는 매개변수의 Idx와 반환하는 idx가 같은 경우만 유일한 문자라는 것만 알아두자.

```javascript
function solution(s) {
  let res = "";
  for (let i = 0; i < s.length; i++) {
    if (s.indexOf(s[i]) === i) res += s[i];
  }
  return res;
}
```

## 3차 풀이 \_Set 자료형 사용

set은 알아서 중복을 제거해준다. 이를 이용하자.
문자열을 분리해서 캐릭터로 만들어준다.
그리고 중복 제거후 -> join 한다.

Set 자료형은 애초에 전달받은 값이 이터러블 객체인지 확인하고, 이터러블 객체라면 요소 하나씩 순회하면서 중복을 제거한다. 그래서 new Set([...s])대신 new Set(s)를 해도 되는것이다. 문자열도 이터러블 객체이기 때문에 가능한 것이다.

```javascript
function solution(s) {
  const set = new Set(s);
  return [...set].join("");
}
```
